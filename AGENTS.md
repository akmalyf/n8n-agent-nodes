# Agent Instructions

## Package Manager
- Use npm; `package-lock.json` is present.
- Install with `npm ci` for reproducible installs, or `npm install` when intentionally updating the lockfile.

## Commands
| Task | Command |
|------|---------|
| Build / typecheck | `npm run build` |
| Watch TypeScript | `npm run dev` |
| Clean generated output | `npm run clean` |

## Verification
- There are no configured `test`, `lint`, or formatter scripts; `npm run build` is the canonical local check.
- `dist/` is generated by `tsc` from `src/` and is ignored by git.

## n8n Package Wiring
- n8n discovers the package from `package.json` `n8n.credentials` and `n8n.nodes`; keep these paths aligned with compiled `dist/**` outputs.
- Public exports are in `src/index.ts`; use `.js` suffixes in TypeScript exports/imports when matching the existing pattern.
- Main node implementation: `src/nodes/AIAgent/AIAgent.node.ts`.
- Credential type name is `aiApi` in `src/credentials/AiApi.credentials.ts`; the node credential reference must match it.

## Runtime Notes
- The node uses AI SDK v4 packages (`ai`, `@ai-sdk/openai`, `@ai-sdk/anthropic`) and n8n peer dependencies (`n8n-core`, `n8n-workflow`).
- Memory is in-process only via the singleton `agentRuntime` in `src/shared/AgentRuntime.ts`; do not assume persistence across n8n restarts or separate worker processes.
- The `custom` provider exists in credentials, but node execution currently branches only between `anthropic` and the OpenAI-compatible path.

## Commit Attribution
AI commits MUST include:
```
Co-Authored-By: (the agent's name and attribution byline)
```
